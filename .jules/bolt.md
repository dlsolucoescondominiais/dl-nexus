## 2024-05-24 - N+1 query problem with Google Drive recursive folder lookup
**Learning:** Found a major N+1 query problem inside the codebase's architecture (`execution/agente_zelador.py` and `execution/agente_arquivista.py`). When the AI agents dynamically process incoming files, they issue network API calls to the Google Drive API to look up if the parent folders (Category, Year, Month) exist and if not create them. Since many files land in the same folders, this executes a redundant query for every file in the loop, creating a huge bottleneck, unnecessary network latency and risking severe rate-limiting by Google.
**Action:** Always implement an in-memory dictionary cache (e.g., `_folder_cache = {}`) when working with external APIs (like Google Drive) to store the retrieved object IDs. This reduces O(N) API calls to O(1) for repeated destination paths.
