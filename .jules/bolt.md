## 2024-05-24 - [Avoid N+1 queries using in-memory caches on API clients]
**Learning:** During batch processing of files via external APIs (such as Google Drive API in `execution/agente_zelador.py` and `execution/agente_arquivista.py`), the codebase was unnecessarily fetching folder IDs on every single file loop (the N+1 query problem). This is a known performance anti-pattern and often leads to rate limits and high response times when multiple resources share the same target folder IDs.
**Action:** Always implement an in-memory dictionary cache (e.g., `_folder_id_cache` keyed by parent_id and name) inside helper functions that repeatedly lookup or create external resources during loops. This applies specifically to Python agent scripts making Google Drive or other external API calls.
