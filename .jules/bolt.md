## 2024-07-09 - [Optimize Derived State KPI Calculation in Real-time Component]
**Learning:** In React components consuming real-time data from Supabase websockets, using `useState` to manually synchronize secondary metric data (like KPIs) triggers redundant re-renders and creates multiple sources of truth. Furthermore, relying on multiple sequential `.filter().length` array passes (O(M*N)) is inefficient compared to loop fusion (O(N)) for calculating multiple scalar metrics from a single large dataset.
**Action:** Replaced secondary state (`kpis`, `setKpis`) with a `useMemo` dependency array tracking the primary dataset (`leads`). Consolidated three `.filter()` passes into a single `for` loop traversal over the `leads` array, significantly reducing computational overhead during real-time updates.
