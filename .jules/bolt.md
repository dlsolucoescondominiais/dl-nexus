## 2026-05-29 - Prevent Event Loop Blocking in FastAPI Routes
**Learning:** In FastAPI, if you use `async def` for a route handler but perform synchronous I/O operations inside it (like using the synchronous `requests` library or the synchronous OpenAI SDK), it blocks the entire event loop. This causes severe performance degradation and latency spikes because other requests cannot be processed concurrently.
**Action:** When a route handler must execute synchronous, blocking I/O operations (like `requests.post()` in marketing routes or `openai.chat.completions.create()` in AI agent integrations), drop the `async` keyword and use a standard `def`. This allows FastAPI to transparently offload the execution to an external threadpool, preventing the main event loop from blocking.
