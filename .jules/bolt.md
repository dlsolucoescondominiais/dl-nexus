## 2024-05-13 - FastAPI Performance Anti-pattern: Synchronous I/O in `async def` routes
**Learning:** Routes performing synchronous I/O (like `requests.post` or calling synchronous SDK clients, e.g., OpenAI API) must be defined with `def` rather than `async def`. If declared as `async def`, these synchronous operations completely halt the server's single-threaded event loop. Using `def` automatically offloads them to an external thread pool, preventing the server from blocking and dramatically improving throughput for these endpoints.
**Action:** When creating new routes that rely on synchronous blocking calls (e.g. `requests`, synchronous database clients, or SDKs without async support), ALWAYS define the route using `def`. Only use `async def` if the route exclusively uses asynchronous, non-blocking I/O.
