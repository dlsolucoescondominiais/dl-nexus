## 2024-05-24 - Google Drive API Performance Bottleneck & In-Memory Folder Caching
**Learning:** The `execution/agente_zelador.py` and `execution/agente_arquivista.py` scripts query the Google Drive API for folder existence and creation. When processing a batch of files in a loop, this leads to numerous redundant API calls (e.g., repeatedly searching for the same year/month folder). This redundancy introduces significant network latency and increases the risk of hitting Google API rate limits (e.g., 429 Too Many Requests). This is a codebase-specific performance pattern since it relies heavily on synchronous file routing using Google Drive structure as the primary data store.
**Action:** Implemented an in-memory dictionary cache (`_folder_cache`) keyed by `parent_id` (or `ARCHIVE_FOLDER_ID`) and `nome_pasta` within the `obter_ou_criar_pasta` function. This avoids redundant external API calls during batch processing, immediately returning the cached `folder_id` and significantly reducing execution time. Next time, always check if repeated external API queries for static or slow-changing resources within the same execution context can be avoided with a simple dictionary cache.
