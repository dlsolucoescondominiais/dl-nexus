## 2024-04-27 - FastAPI Event Loop Starvation with Synchronous I/O
**Learning:** In FastAPI, routes or methods that are defined with `async def` run directly on the main event loop. If these routes contain synchronous, blocking I/O operations (like `requests.post` or synchronous `openai` API calls), they will block the entire event loop, preventing other concurrent requests from being processed. Conversely, routes defined with `def` are run by FastAPI in an external thread pool, preserving event loop responsiveness.
**Action:** When working on FastAPI backends, I must carefully audit `async def` routes to ensure they only contain non-blocking, awaited I/O operations (e.g., `httpx` or `AsyncOpenAI`). If a route inherently requires a synchronous blocking library (like `requests`), it must be defined with `def` rather than `async def` to offload the blocking operation to a thread pool.
