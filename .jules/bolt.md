## 2024-05-24 - ThreadPoolExecutor for concurrent batch processing with strict rate limits
**Learning:** Initial attempt to parallelize API batch jobs with `time.sleep` stagger failed because threads sat completely idle, waiting for their turn before even starting IO prep. Proper optimization requires parallelizing all independent IO-bound pre-work (e.g., Gemini API, Drive API, Video rendering), and only locking (`threading.Lock()`) around the specific rate-limited external calls, computing dynamic sleep deltas against a shared `_last_publish_time`.
**Action:** Always wrap concurrent task dispatch loops. Separate pre-computation from rate-limited API dispatch, wrapping only the API dispatch in thread-safe locks. Use `array[i] = future.result()` to maintain state alignment and fast-fail behavior.
