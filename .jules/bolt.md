## 2024-05-03 - FastAPI Event Loop Starvation with Sync I/O
**Learning:** Routes performing synchronous I/O (like `requests.post` or OpenAI sync client) must be defined with `def` rather than `async def`. This allows FastAPI to execute them in an external thread pool, preventing them from blocking the single-threaded asyncio event loop. If declared as `async def`, these synchronous operations completely halt all other concurrent request processing in the server.
**Action:** Always check the internal implementations of route handlers and their agent dependencies. If they contain blocking network calls (e.g., standard `requests` or synchronous SDKs), either refactor to use async clients (e.g., `httpx.AsyncClient`) or define the FastAPI route as `def` to ensure the application remains responsive under load.
