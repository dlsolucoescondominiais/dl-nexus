## Como Usar Este Checklist

- ✅ Cada item deve ser verificado individualmente.
- ❌ **Qualquer item reprovado** bloqueia a publicação.
- O KILLCRITIC (Code Node) aplica estas verificações automaticamente.
- Para auditorias manuais, use este checklist como referência.
- **Tolerância zero** para violações de privacidade, segurança e compliance.

## 1. Texto e Copy

### 1.1 — Tom e Abordagem

- [ ] Texto **NÃO** acusa o leitor diretamente?
- [ ] **NÃO** usa "você/seu/sua" ligado a risco, falha, insegurança ou atributo sensível?
- [ ] Tom é **técnico, preventivo e comercial moderado**?
- [ ] Linguagem é profissional, sem gírias ou informalidade excessiva?
- [ ] Texto transmite autoridade técnica sem arrogância?

### 1.2 — Promessas Proibidas

- [ ] **NÃO** promete resultado garantido?
- [ ] **NÃO** promete economia garantida?
- [ ] **NÃO** promete segurança total?
- [ ] **NÃO** promete prazo sem avaliação técnica prévia?
- [ ] **NÃO** garante prevenção total de qualquer sinistro?
- [ ] **NÃO** usa percentuais de economia sem disclaimer?

### 1.3 — Termos Proibidos

- [ ] **NÃO** usa "blindado 24h"?
- [ ] **NÃO** usa "nunca mais terá problema"?
- [ ] **NÃO** usa "risco de morte"?
- [ ] **NÃO** usa "tragédia"?
- [ ] **NÃO** usa "perigo total"?
- [ ] **NÃO** usa "urgente!!!" ou variações sensacionalistas?
- [ ] **NÃO** usa "chocante"?
- [ ] **NÃO** usa sensacionalismo de qualquer tipo?

### 1.4 — Termos Obrigatórios e Padronização

- [ ] Usa **"Avaliação Técnica"** (não "visita técnica", não "orçamento grátis")?
- [ ] Usa **"tempo de resposta para chamados"** quando fala de SLA?
- [ ] Linha DL está **correta para o tema** do dia (conforme matriz semanal)?
- [ ] Hashtags estão alinhadas com a linha DL do dia?

### 1.5 — CTA (Call to Action)

- [ ] CTA é seguro? (Avaliação Técnica, chame no Direct, envie mensagem, fale com a DL)?
- [ ] **NÃO** usa "compre agora"?
- [ ] **NÃO** usa "últimas vagas"?
- [ ] **NÃO** usa "só hoje"?
- [ ] **NÃO** usa "orçamento grátis"?
- [ ] **NÃO** usa "não perca"?

### 1.6 — Informações Falsas e Associações Indevidas

- [ ] **NÃO** diz que a DL é parceira da Meta?
- [ ] **NÃO** diz que a DL é parceira do Facebook?
- [ ] **NÃO** diz que a DL é parceira do Instagram?
- [ ] **NÃO** diz que a DL é parceira do WhatsApp?
- [ ] **NÃO** inventa convênio com Guarda Municipal?
- [ ] **NÃO** inventa convênio com qualquer força policial?
- [ ] **NÃO** inventa integração oficial com órgãos públicos?
- [ ] **NÃO** inventa integração oficial com Defesa Civil?
- [ ] **NÃO** inventa integração oficial com Corpo de Bombeiros?
- [ ] **NÃO** inventa integração oficial com Prefeitura?
- [ ] **NÃO** afirma compatibilidades não comprovadas entre sistemas?

### 1.7 — Originalidade e Direitos

- [ ] **NÃO** copia notícia textualmente?
- [ ] **NÃO** reproduz parágrafos de fontes jornalísticas?
- [ ] **NÃO** identifica condomínios, endereços ou vítimas de notícias?
- [ ] Conteúdo é original e inspirado (não copiado)?

### 1.8 — Jargão Interno

- [ ] **NÃO** usa "B2B" em texto público?
- [ ] **NÃO** usa "SLA" sem traduzir para "tempo de resposta para chamados"?
- [ ] **NÃO** usa jargão técnico de TI incompreensível para síndicos?

### 1.9 — Qualidade Geral

- [ ] Gramática está correta?
- [ ] Ortografia está correta?
- [ ] Pontuação está adequada?
- [ ] Texto tem tamanho adequado para a plataforma (não muito longo para feed)?
- [ ] Emojis usados com moderação e pertinência?

## 2. Imagem e Mídia

### 2.1 — Estilo Visual

- [ ] Imagem é **fotográfica, realista, profissional**?
- [ ] Iluminação é natural ou de estúdio (não artificial/render)?
- [ ] **NÃO** tem aparência de videogame?
- [ ] **NÃO** tem aparência de 3D genérico?
- [ ] **NÃO** tem aparência de render artificial?
- [ ] Cores são naturais e equilibradas?
- [ ] Composição é limpa e profissional?

### 2.2 — Elementos Proibidos na Imagem

- [ ] **NÃO** tem texto dentro da imagem (nenhuma palavra, frase, número ou legenda)?
- [ ] **NÃO** tem logotipo Meta / Facebook / Instagram / WhatsApp?
- [ ] **NÃO** tem logotipo de nenhuma marca (DL Soluções incluída)?
- [ ] **NÃO** tem logotipo de fabricantes de equipamentos?
- [ ] **NÃO** tem pessoas identificáveis ou rosto em destaque?
- [ ] **NÃO** tem placa de carro visível?
- [ ] **NÃO** tem número de apartamento visível?
- [ ] **NÃO** tem fachada identificável de cliente?
- [ ] **NÃO** tem uniforme com marca falsa?
- [ ] **NÃO** tem cenas de pânico?
- [ ] **NÃO** tem incêndio real?
- [ ] **NÃO** tem acidente?
- [ ] **NÃO** tem vítimas?
- [ ] **NÃO** tem tragédia?

### 2.3 — Formato e Técnica

- [ ] Formato correto para a plataforma destino?
  - Feed: 1080×1080 (1:1)
  - Stories/Reels: 1080×1920 (9:16)
  - Carrossel: 1080×1080 por slide
- [ ] Resolução mínima de 1080px no menor lado?
- [ ] Arquivo é JPEG ou PNG?
- [ ] Tamanho do arquivo dentro do limite da Meta (máx. 8MB para imagens)?

### 2.4 — Direitos e Origem

- [ ] Tem direito de uso da mídia (gerada por IA = sem restrição)?
- [ ] **NÃO** usa `picsum.photos` para post real?
- [ ] **NÃO** usa placeholder genérico para publicação real?
- [ ] Imagem foi gerada especificamente para este post?

## 3. Meta API e Publicação

### 3.1 — Autenticação

- [ ] Token é **Page Access Token** (não User Token)?
- [ ] Token **NÃO** está hardcoded no JSON do workflow?
- [ ] Token **NÃO** está exposto em log de execução?
- [ ] Token **NÃO** está em variável de ambiente com nome genérico?
- [ ] Token é referenciado via `{{$env.META_PAGE_ACCESS_TOKEN_DL}}`?
- [ ] Token tem permissões necessárias (`pages_manage_posts`, `pages_read_engagement`, `instagram_basic`, `instagram_content_publish`)?
- [ ] Token não está expirado?

### 3.2 — Endpoints Facebook

- [ ] Facebook usa endpoint `/PAGE_ID/feed` para posts de texto?
- [ ] Facebook usa endpoint `/PAGE_ID/photos` para posts com imagem?
- [ ] `PAGE_ID` vem de `{{$env.META_PAGE_ID_DL}}`?
- [ ] Versão da API é v21.0 ou superior?

### 3.3 — Endpoints Instagram

- [ ] Instagram usa endpoint `/IG_ID/media` para criar container?
- [ ] Instagram usa endpoint `/creation_id/media_publish` para publicar?
- [ ] `IG_ID` vem de `{{$env.META_IG_BUSINESS_ID_DL}}`?
- [ ] Fluxo de 2 etapas (criar → publicar) está implementado?
- [ ] Versão da API é v21.0 ou superior?

### 3.4 — URL de Imagem (Instagram)

- [ ] Instagram tem `image_url` pública HTTPS válida?
- [ ] `image_url` abre sem login (testar em aba anônima)?
- [ ] `image_url` retorna `Content-Type: image/jpeg` ou `image/png`?
- [ ] `image_url` **NÃO** é Google Drive privado?
- [ ] `image_url` **NÃO** redireciona para página de login?
- [ ] `image_url` **NÃO** é `picsum.photos` ou placeholder?
- [ ] `image_url` responde em menos de 5 segundos?

### 3.5 — Validação de Publicação

- [ ] Publicação retornou `post_id` real (Facebook)?
- [ ] Publicação retornou `media_id` real (Instagram)?
- [ ] **NÃO** está marcando publicação como concluída sem `post_id`/`media_id`?
- [ ] **NÃO** confunde workflow ativo com publicação real?
- [ ] Registro no Supabase contém `post_id`/`media_id` válido?
- [ ] Notificação Telegram confirma publicação com link?

## 4. Compliance e Segurança

### 4.1 — Proteção de Credenciais

- [ ] **NÃO** expõe `META_PAGE_ACCESS_TOKEN_DL` em nenhum output?
- [ ] **NÃO** expõe `META_APP_SECRET` em nenhum output?
- [ ] **NÃO** expõe `SUPABASE_SERVICE_ROLE_KEY` em nenhum output?
- [ ] **NÃO** expõe `IMAGE_API_KEY` em nenhum output?
- [ ] **NÃO** expõe `TELEGRAM_BOT_TOKEN` em nenhum output?
- [ ] **NÃO** commita token em repositório Git?
- [ ] **NÃO** loga token em console/debug?
- [ ] Todas as credenciais são referenciadas via `{{$env.VARIAVEL}}`?

### 4.2 — Dependências Externas

- [ ] **NÃO** usa Manus.IA?
- [ ] **NÃO** depende de `MANUS_API_KEY`?
- [ ] **NÃO** faz chamadas para APIs do Manus?
- [ ] Landing page existe e funciona (se referenciada no post)?
- [ ] Links externos no post são válidos e acessíveis?

### 4.3 — LGPD e Privacidade

- [ ] Nenhum dado pessoal de condôminos exposto?
- [ ] Nenhuma imagem de câmera real de condomínio cliente?
- [ ] Nenhum endereço real de cliente mencionado?
- [ ] Nenhum nome de síndico mencionado sem autorização?

### 4.4 — Políticas da Plataforma Meta

- [ ] Conteúdo não viola Community Standards do Facebook?
- [ ] Conteúdo não viola Community Guidelines do Instagram?
- [ ] Conteúdo não é classificável como "Engagement Bait"?
- [ ] Conteúdo não é classificável como "Misleading Content"?
- [ ] Conteúdo não é classificável como "Clickbait"?
